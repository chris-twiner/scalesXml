16 /**
17  * Most of the JAXP, STAX and DOM apis don't mention much about
18  * thread safety but the dominant xerces is completely unsafe with regards
19  * to threads.  In fact only the 1.4.2 apis mention anything about thread safety
20  * expectations.
21  *
22  * The process of getting a factory is very expensive and are not thread safe (you can't trust it to create a document in parallel).  See MUSE 270 for an example of why.
23  *
24  * As such we must use a pool for all handling.  A thread/piece of code
25  * grabs a factory from the pool, creating if necessary.
26  *
27  * To aid the user the parsing code uses the Loaner interface (for SAX and DOM factories) and uses the Pool interface directly for Pull Parsers, as using them does not imply a given scope.
28  *
29  * By default no validating is performed
30  */

172     val p = System.getProperty("scales.traxSourceShouldSerialize")
173     if (p ne null)
174       p.toBoolean
175     else {
176

178 
179       val t = TransformerFactory.newInstance.newTransformer
180       t.getClass.getName == "org.apache.xalan.transformer.TransformerIdentityImpl"
181     }
182   }
183 
184   /**
185    * If newSchema is called should we serialize the source?  Sun JDK does trax but not validation for StAXSource :<
186    * 
187    * JDK 1.7 shares the StAX love though.
188    *
189    * If scales.newSchemaShouldSerialize is defined its value is used.
190    *
191    * Otherwise we examine the implemenation version of jaxp schemafactory, 1.5 and 1.6 need to serialize, 1.7 (and hopefully above ^_^) don't.
192    *
193    * Assuming non sun 1.8 and above come ready to validate.
194    */ 
195   lazy val newSchemaShouldSerialize = {
196     val p = System.getProperty("scales.newSchemaShouldSerialize")
197     if (p ne null)
198       p.toBoolean
199     else {
200       val fac = SchemaFactory.newInstance(javax.xml.XMLConstants.W3C_XML_SCHEMA_NS_URI)
201       val ver = fac.getClass.getPackage.getImplementationVersion
202 
203       if (ver ne null) {
204         val pre = ver.substring(0, 3)
205 
206         if (pre == "1.5" || pre == "1.6")
207           true
208         else
209           false
210       } else
211         false // null for 1.8 and above, otherwise use the property
212     }
213   }
